# VRCSM GUI↔API Contract Audit (2026-07-09)

## Method

The audit fanned out one verification pass per bridge translation unit (21 per-domain bridges under `src/host/bridges/` plus `PluginBridge`), building a fact-base of 185 registered IPC handlers, 128 frontend call sites, and 178 methods in `AsyncMethodSet`, then adversarially cross-checked each handler's request/response/event shape and error convention against its TypeScript consumer (`web/src/lib/ipc.ts` facades, the domain facades, and the calling pages/hooks). Every candidate finding was re-opened at the cited lines on both sides of the boundary — reading the actual C++ handler body and the actual FE branch/validator — before being marked CONFIRMED or PLAUSIBLE; findings that did not survive that read were rejected. Coverage was assessed separately against the three smoke matrices (`interaction-smoke.test.tsx`, `tests/smoke/helpers.ts`, `pages-smoke.test.tsx`) and the five registered result-shape validators.

## Verdict

**Grade: B-.** The GUI↔API contract is structurally sound — the plugin origin sandbox is enforced per-method and layered, most mutators use a stable `IpcException`+code pattern, user input is percent-encoded into the majority of REST paths, and the destructive `SafeDelete`/`Migrator` paths are genuinely hardened. But it is dragged down by one **critical** SSRF hole in the lyrics proxy, a cluster of **high**-severity availability/correctness bugs (auth logout-flap on any 429, friends list masking outages as "0 friends", OSC float type-loss, a Registry-tab white-screen, a destructive junction repair that skips the running-VRChat guard), and thin end-to-end coverage on exactly the irreversible flows (migrate, avatar delete, SteamVR repair, self-update) where a silent contract drift would be most damaging. Nothing here is a cross-origin auth escalation, but several are real user-facing regressions.

## Findings (severity-ranked)

### CRITICAL

**[CRITICAL] LyricsProxy SSRF rail is bypassable via HTTP redirects and DNS names** — `lyrics.fetch` — security — GUI `web/src/lib/lyrics.ts:280` — API `src/core/LyricsProxy.cpp:205, :259-261, :108-162` — The host check runs `IsBlockedProxyHost` once against the *original literal-IP* host only, then sets `WINHTTP_OPTION_REDIRECT_POLICY_ALWAYS` with no per-hop callback, so (a) a public DNS name resolving to `127.0.0.1`/`169.254.169.254` (e.g. `localtest.me`) passes the literal-IP filter and reaches an internal endpoint, and (b) a `302` to an intranet host (incl. https→http downgrade) is followed unvalidated. — Fix: register a `WINHTTP_STATUS_CALLBACK` on `WINHTTP_CALLBACK_STATUS_REDIRECT`, re-run `IsBlockedProxyHost` per hop rejecting non-https hops, and resolve the hostname to block loopback/private/link-local addresses before connect. — **CONFIRMED**

## Domain health

- **ApiBridge (50)** — Broadly well-built (consistent codes, percent-encoding, field allowlists, SSRF host allowlist on image paths); material issues are behavioral: empty-success masking on friends/moderations + redundant probe, seven `runtime_error` handlers, and encoding inconsistencies.
- **AuthBridge** — Login secret-handling is disciplined, but status/user/verify2FA collapse transient failures into logged-out, driving a logout-flap and a malformed-AuthStatus-on-2FA path.
- **UpdateBridge** — Recently hardened (mandatory SHA fail-closed, path canonicalization, GitHub host pin); residual issues are a mis-tiered blocking download and low-severity downloader defense-in-depth gaps.
- **DatabaseBridge** — Strong on bound placeholders + allowlisted DELETEs; issues are one unbounded co-presence scan, one history.clear shape drift, four dead handlers, and non-uniform limit clamps.
- **PipelineBridge (21)** — Solid transport/lifecycle discipline; one user-facing OSC float type-fidelity regression plus medium drifts (listen bind, Discord id, orphan screenshots.new).
- **SettingsBridge + ShellBridge** — Destructive SteamVR/reset paths are backup-first and traversal-guarded, but the `{error}`-as-success convention crashes the Registry tab and hides write failures; `shell.openUrl` blocks the UI thread; plugin fs surface is over-exposed.
- **RuleBridge + EventBridge (13)** — Thin correct pass-through with aligned happy-path shapes; latent robustness gaps (type-unsafe update, silent no-op success, 32-bit id truncation, missing confirm).
- **CacheBridge / MigrateBridge** — `SafeDelete`/`Migrator` are notably hardened, but `junction.repair` skips the ProcessGuard its siblings enforce.
- **LyricsBridge** — Contract-aligned but carries the one critical SSRF hole plus a referer header-injection gap.
- **VrDiagBridge** — `vr.audio.switch` ignores its role param and swallows every HRESULT.
- **ScreenshotBridge** — open/folder/delete correctly contained; the two metadata handlers skip the same guard and are dead.
- **HwBridge / RadarBridge / MusicBridge / SearchBridge / LogsBridge / VectorBridge** — Shapes align cleanly; only dead-code/orphan-subscriber cleanups.
- **PluginBridge** — Sandbox enforcement is the strongest part of the surface; the audit fact-base omitted it (handlers.tsv gap) and `plugin.rpc` is untested.

2. **`plugin.rpc` dead in mock + both plugin routes (`/plugins/:id`, `/p/:pluginId/*`) untested** — throws `mock_not_implemented` in dev.
